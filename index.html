A list of deployed projects you can peruse!

1:Gweather: an LGBTQIA+ website built around the concept of Provincetown RI and why it is a renown haven for people who are part of the community. This was built using HTML/CSS/JS front end that accesses data from a public API using JSON-server. This project returns a collection of objects with each object having multiple attributes. This is a single page with no redirectos or reloads. It incorporates event listeners and iterations. I had a lot of fun figuring out CSS for this project.
2: PotterHub: a compendium of Harry Potter Characters and spells that you can use to find niche information on the movies, books, characters and spells as well as have the ability to create your own spells and characters! This was built using a single page application with Vite. This project contains several components, multiple client side routes, and a nav bar.
3: CLI Event Planner:The Performance Event Planner is a Python-based command-line interface application designed to help: local artists manage events, tours and build a fan following as well as find venues to perform at based on details.; It Helps local venues find artists to perform for them and to see data on attendees to help cater events to popular demand. Venues can also create events and tours to promote them. Lastly it helps attendees to find local events/tours in the area, follow artists to different shows or follow venues to see their events and who is performing at them.
I personally created this to track my singing performances and friends and family that helped support me along the way! The application allows users to GET, POST, PATCH and DELETE based on their intended use of the database All through the terminal! I became so passionate about this project and determined to be able to share it with people who aren't technically inclined so i let this project guide me into the next phase of learning.
4: PRISM: Prism is a fully formed application with all the previous projects functionality and more! This uses a flask API backend with a React Front-end. We used several models, one-to-many relationships, many-to-many relationships full CRUD actions, Data type Validations, multiple client side routes, complete Authentication allowing access and permissions based on user type using cookies; including an ADMIN role that has higher access to useful metrics on users and data, with a tight security utilizing hashed passwords.
